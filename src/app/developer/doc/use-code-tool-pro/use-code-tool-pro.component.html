CGT进阶主要讲述对生成组件及指令的使用。

如上所示，CGT生成代码包含2个服务，2个组件，3个指令。下面以DemoOrder为例列出这些内容：

服务 DemoOrderFieldInfoService
DemoOrderService
组件 app-list-demo-order
app-view-demo-order-base-info
指令 [appCommonEdit]
[selectDemoOrder]
[appCommonView]

服务

  1. 服务 DemoOrderFieldInfoService：

    2. 该服务有以下几点功能，

    1. 配置主表字段信息，诸如：字段是否参与查询，是否参与排序，数据格式化方式等,详情请参见类FieldInfo的定义；
    2. 提供以下继承自其父类（FieldInfoService）的功能。

    下面讲解FieldInfoService中的常用属性及方法：

    属性
    deleteField string， 如果该表中的数据使用的是标记删除，则需要配置某个具体的字段为标记删除的字段，例如 'isDel'为标记删除字段，则配置 deleteField = 'isDel'即可
    方法
    get entityName() 获取操作的实体名，抽象方法，这里主要指真实的实体
    get queryEntityName() 获取查询实体名，抽象方法，这里主要指视图名（如果记得之前提到过的使用视图去生成代码，则在这里就需要去掉生成的实体名和视图名后面多余的View）
    get _fieldInfo() 获取字段配置信息
    get uiLabels() 获取中英文对照map
    getEntityPK(entity) 根据实体对象，获取主键对象
    getEntityPKValue(entity) 根据实体对象，获取主键的值
    loadSelectDataList() 初始化当前服务管理的所有下拉框数据.
    validateGenericValue(data) 验证数据是否可以为空
    formatFieldValue(obj, fieldInfo) 格式化数据，将date和dateTime类型的数据格式化成'yyyy-MM-dd'和'yyyy-MM-dd HH:mm:ss'格式
    getOne(submitQueryData): Observable 通过主键查询条件获取一条数据
    getPageData(submitQueryData): Observable 通过查询条件获取已进行分页处理的数据
    genericSave(fieldMap): Observable 保存一条数据
    genericSaveAll(dataList): Observable 批量保存多条数据
    deleteData(PK): Observable 根据主键删除数据
  2. 服务 DemoOrderService：

    3. 该服务有以下几点功能，

    1. 该服务类是实际业务的处理类，与后台数据交互的业务处理都应在该服务类中进行操作；
    2. 业务是否参与审批的配置类，平台流程引擎使用的 activi 工作流，使用中只要在 processInstanceKey 方法中配置对应的 activi 流程编号，即可参与流程审批；
    3. 提供以下继承自其父类（CommonService）的功能。

    下面讲解CommonService中的常用属性及方法：

    属性
    hasAttachment boolean，配置界面是否有上传附件的功能，true为有，false为没有改功能，默认值为false。
    方法
    get processInstanceKey() 获取流程编号
    get hasFlow() 根据 processInstanceKey 方法返回的流程编号，判断是否参与了审批，有编号则认为参与审批，反之，则没有参与。

组件

  1. 组件 app-list-demo-order ：

    2. 该组件继承CommonDataTable是CURD主界面（该组件主要作为一个单独模块使用，如果需要作为其他界面的一部分使用，请单独使用CGT工具生成代码放置到相应模块使用）。

    使用标签app-list-demo-order
    导入模块ListDemoOrderModule
    依赖模块
    依赖服务DemoOrderFieldInfoService
    订单列表

    下面讲解 ListDemoOrderComponent 及其父类 CommonDataTable中的常用属性及方法：

    属性
    isQueryCollapsed boolean，查询框是否折叠，默认为false
    isOpratorCollapsed boolean，数据框是否折叠，默认为false
    hasBackButton boolean，界面是否有返回按钮，默认为false
    showDataTableOnly submitQueryData，当前组件是否仅显示表格，不显示查询框，默认为false
    queryData object，输入属性，传入默认查询条件
    dataList array，页面展示的数据集
    submitQueryData object，提交到后台用于查询的数据
    pageData object，提交到后台用于分页处理的数据
    方法
    get isAllChecked()返回是否全选
    get checkedNum()返回选中数量
    get selectedList(): Array返回选中数据列表
    get selectedPK(): Array返回选中数据主键列表
    beforeInit()类初始化之前执行，该方法体为空，子类可按需重写
    beforeQuery()进行查询之前执行，用于前置数据处理和查询条件拼装，该方法体为空，子类可按需重写
    afterDataLoaded()数据加载之后执行，每次查询完都执行（用于在前台进行少量后置数据处理），该方法体为空，子类可按需重写
    afterDataSaved(savedValue: any)数据保存后调用的方法（用于在数据保存后的操作处理），该方法体为空，子类可按需重写
    afterDataDeleted(PK: any)数据删除之后执行（用于在数据删除后的操作处理），该方法体为空，子类可按需重写

指令

  1. 指令 [appCommonEdit]：

    2. 该组件继承 CommonEditComponent 和 CommonComponent 是数据新增及修改界面，新增修改通过指令输入值[Pk]确定([PK]有值则为编辑，没有则为新增，详见右侧两个按钮)。

    使用指令[appCommonEdit]
    导入模块CommonEditModule
    依赖模块
    依赖服务DemoOrderService,DemoOrderFieldInfoService

    下面讲解 CommonEditComponent 和 CommonComponent 中的常用属性及方法：

    属性
    defaultAddValueobject，输入属性，新增数据时的默认数据
    PKobject，输入主键，有主键则判断为新增，没有则是编辑
    handleTaskComponentstring，输入属性，任务办理界面组件
    dataSavedEvent事件，数据保存后触发
    handleTaskEvent事件，启动任务处理事件
    handleTaskDoneEvent事件，任务处理完成事件
    closeViewEvent事件，关闭页面事件,当组件被用作弹出窗口的时候使用
    方法
    commitToServer()若需要自己保存数据（用于进行多表的数据保存操作），请在子类重写此方法
    afterGetOne()在获取编辑数据后执行，该方法体为空，子类可按需重写
    beforeValidateAndFormat()在验证与格式化要提交的数据之前执行，该方法体为空，子类可按需重写
    beforeSubmit(validate: EntityFieldValidate)提交数据之前执行（用于前置数据的处理），该方法体为空，子类可按需重写
    afterSave(savedValue: any)数据保存成功之后执行（用于数据保存后的操作），该方法体为空，子类可按需重写
    afterInit()页面加载完成后触发，该方法体为空，子类可按需重写
    formatDuring()格式化时间间隔，用户将时长毫秒格式化成'天/时/分/秒'的可读间隔
